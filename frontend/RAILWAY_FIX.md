# Railwayデプロイ修正

## 問題

Railwayサーバにデプロイした資材は起動してもログイン画面しか出力されません。

## 原因

ルートパス（/）がログインページにリダイレクトされている可能性があります。

## 対応内容

### 1. ルートパスの修正

`app/page.tsx`を修正して、ダッシュボードを直接表示するように変更しました。

### 2. 認証チェックの無効化

`app/dashboard/page.tsx`の認証チェックは既にコメントアウトされています。

### 3. Railwayへの再デプロイ

修正したコードをRailwayに再デプロイする必要があります。

## 確認事項

1. **Railwayのデプロイ設定**
   - フロントエンドが正しくデプロイされているか確認
   - ビルドコマンドが`npm run build`になっているか確認
   - スタートコマンドが`npm start`になっているか確認

2. **環境変数**
   - `NEXT_PUBLIC_API_BASE_URL`が正しく設定されているか確認

3. **ルーティング**
   - ルートパス（/）がダッシュボードを表示するように設定されているか確認

## デプロイ手順

1. コードをコミット
2. Railwayにプッシュ
3. Railwayが自動的に再デプロイを開始
4. デプロイ完了後、ルートURLにアクセスしてダッシュボードが表示されることを確認

## 確認URL

- **Railway**: https://cms-production-4115.up.railway.app

## トラブルシューティング

### ログイン画面が表示される場合

1. ブラウザのキャッシュをクリア
2. シークレットモードでアクセス
3. Railwayのログを確認してエラーがないか確認

### 404エラーが発生する場合

1. Railwayのビルドログを確認
2. `next.config.js`の設定を確認
3. ルーティング設定を確認
