# 開発サーバー再起動手順

## 現在の状況
- ポート3002で開発サーバーが起動している
- しかし、静的ファイル（CSS、JS）が404エラーになっている
- 開発サーバーを再起動する必要があります

## 解決手順

### ステップ1: 開発サーバーを停止

現在実行中の開発サーバーを停止：
- ターミナルで **Ctrl + C** を押す
- または、プロセスを終了（実施済み）

### ステップ2: キャッシュをクリア（実施済み）

`.next`ディレクトリを削除しました。

### ステップ3: 開発サーバーを再起動

新しいターミナルで以下を実行：

```bash
cd C:\devlop\cms\frontend
npm run dev
```

これでポート3002で開発サーバーが起動します。

### ステップ4: ブラウザで確認

1. **ブラウザのキャッシュをクリア**
   - Ctrl + Shift + Delete
   - 「キャッシュされた画像とファイル」にチェック
   - 「データを消去」をクリック

2. **シークレットモードでアクセス**
   - Ctrl + Shift + N（Chrome）
   - `http://localhost:3002` にアクセス

3. **開発者ツールで確認**
   - F12で開発者ツールを開く
   - Consoleタブでエラーがないか確認
   - Networkタブでリソースの読み込みを確認

## 確認事項

- [ ] 開発サーバーが正常に起動している
- [ ] ポート3002でアクセスできる
- [ ] 静的ファイル（CSS、JS）が正常に読み込まれている
- [ ] 404エラーが表示されない

## トラブルシューティング

### まだ404エラーが表示される場合

1. **完全なクリーンビルド**
   ```bash
   cd C:\devlop\cms\frontend
   rmdir /s /q .next
   rmdir /s /q node_modules\.cache
   npm install
   npm run dev
   ```

2. **ポートが使用中の場合**
   ```bash
   # ポート3002を使用しているプロセスを確認
   netstat -ano | findstr :3002
   
   # プロセスを終了（PIDを確認してから）
   taskkill /PID <PID> /F
   ```

3. **別のポートで試す**
   ```bash
   npm run dev -- -p 3000
   ```
   アクセス: `http://localhost:3000`
