# SyntaxError: Invalid or unexpected token - 完全修正ガイド

## 🔴 エラー
```
Uncaught SyntaxError: Invalid or unexpected token
```

## ✅ 即座に試す解決方法（優先順位順）

### 方法1: ブラウザのキャッシュを完全にクリア（最も効果的）

1. **Ctrl + Shift + Delete** を押す
2. **「時間の範囲」** を **「全期間」** に設定
3. 以下にチェック：
   - ✅ 閲覧履歴
   - ✅ Cookie と他のサイトデータ
   - ✅ キャッシュされた画像とファイル
4. **「データを消去」** をクリック
5. ブラウザを完全に閉じる
6. ブラウザを再起動
7. `http://localhost:3002` にアクセス

### 方法2: シークレットモードで確認

1. **Ctrl + Shift + N**（Chrome）でシークレットモードを開く
2. `http://localhost:3002` にアクセス
3. エラーが解消されたか確認

### 方法3: 開発サーバーを完全に再起動

1. **現在の開発サーバーを停止**（Ctrl + C）
2. **すべてのNode.jsプロセスを確認**：
   ```bash
   tasklist | findstr node
   ```
3. **必要に応じて終了**：
   ```bash
   taskkill /F /IM node.exe
   ```
4. **キャッシュをクリア**：
   ```bash
   cd C:\devlop\cms\frontend
   rmdir /s /q .next
   rmdir /s /q node_modules\.cache
   ```
5. **開発サーバーを起動**：
   ```bash
   npm run dev
   ```
6. **ビルドが完了するまで待つ**（"Ready" メッセージが表示されるまで）

### 方法4: 完全なクリーンビルド

```bash
cd C:\devlop\cms\frontend

# すべてのキャッシュを削除
rmdir /s /q .next
rmdir /s /q node_modules\.cache

# 依存関係を再インストール
npm install

# 開発サーバーを起動
npm run dev
```

## 🔍 エラーの詳細を確認

### ブラウザの開発者ツールで確認

1. **F12** で開発者ツールを開く
2. **Console** タブを開く
3. エラーメッセージを確認：
   - エラーが発生しているファイル名
   - エラーが発生している行番号
   - エラーの詳細メッセージ

### エラーメッセージの例

```
Uncaught SyntaxError: Invalid or unexpected token
    at main-app.js:123:45
```

この場合、`main-app.js` の123行目45文字目に問題があります。

## 🛠️ よくある原因と解決方法

### 原因1: ブラウザのキャッシュ

**解決方法:**
- ブラウザのキャッシュを完全にクリア（方法1を参照）

### 原因2: ビルドが完了していない

**解決方法:**
- 開発サーバーを起動してから、ビルドが完了するまで待つ
- ターミナルに "Ready" と表示されるまで待つ

### 原因3: ファイルに不正な文字が含まれている

**解決方法:**
- エラーメッセージで特定されたファイルを確認
- ファイルをUTF-8エンコーディングで保存

### 原因4: エンコーディングの問題

**解決方法:**
1. 問題のあるファイルを開く
2. エンコーディングを **UTF-8** に設定
3. 保存
